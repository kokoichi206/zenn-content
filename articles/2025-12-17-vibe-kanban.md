AI コーディングエージェントを活用した開発が当たり前になってきた昨今、**複数の**エージェントを効率よく並列実行・管理したいというニーズが高まっています。

本記事では、自分がここ数ヶ月使っている『Vibe-Kanban』と、最近追加された**激アツ機能・『複数リポジトリ対応』**についてご紹介します。

## Vibe-Kanban とは

まず初めに、[Vibe-Kanban](https://www.vibekanban.com/) とは何かについて簡単に説明します。
Vibe-Kanban は、Claude Code や Codex CLI, GEMINI CLI などの **AI コーディングエージェントを並列実行・管理するツール**です。

主な特徴・特徴は以下のとおりです。

- **並列実行管理**: 複数のエージェントを同時に実行でき、各タスクは独立した git worktree で動作
- **コードレビューツール**: エージェントの変更内容を PR のように差分表示・確認・承認
- **完全ローカル動作**: コードをサーバーに送信せず、ローカルマシンで完結
- **リリース頻度がエグい**: 最近はほぼ[毎日リリース](https://github.com/BloopAI/vibe-kanban/releases)してる
- **GitHub 等との連携が前提**: GitHub Comments を読み取って対応させることが可能

Node.js があれば `npx vibe-kanban` ですぐに始められます。

### AI マネージャーとしての設計思想がアツい

Vibe-Kanban を触っていると、単なるタスク管理ツールではなく、**AI 時代の新しいコーディング方法を我々に提示している**と感じます。

#### そもそも「カンバン」なのか？

Vibe-Kanban という名前ですが、実際に使ってみると**タスク管理がベース**になっていることがわかります。

![](/images/2025-12-17-vibe-kanban/project.png)

- タスクを登録する（AI エージェントの起動）
- タスクが完了したら通知が飛んでくる
- タスクの subtask も作成できる

つまり、**人間がマネージャーとして AI を管理する**という構図です。
これは従来の「AI にコードを書いてもらう」という発想から一歩進んで、**AI をチームメンバーとして扱う**という考え方に近いです。

### "Attempt" という概念

Vibe-Kanban では、AI にタスクを依頼することを **"Attempt"** と呼んでいます。

![](/images/2025-12-17-vibe-kanban/attempts.png)

この命名が示唆するのは、**タスクは投げ直す前提**だということです。

- 1 回目の Attempt でうまくいかなければ、再試行する
- エージェントすら変えて再試行を試みる (Claude Code → Gemini CLI など)
- **Duplicate** 機能で、タスクを複製して独立した Attempt として実行もできる

まるでガチャを引き直すかのように**成功するまで試行を繰り返す**ワークフローが想定されており、我々ユーザーにそれを意識させるデザインになっています。

### 細かい気遣いが効いている

実際に使っていると、開発者目線での細かい気遣いを随所に感じます。

**worktree 作成時の .env コピー**

各タスクは独立した git worktree で実行されますが、その際に**指定したファイル(.env に限らない)を自動でコピー**してくれます。

**コード差分ページ内の GitHub PR 作成ボタン**

エージェントが生成したコードの**差分表示ページ**に、GitHub PR 作成ボタンが配置されています。

これもまた、**「AI エージェントの時代においても人間のレビューは必要なので、それをちゃんと確認してから PR を作ろうね」という設計の意図**が感じられる UI となっており非常に参考になります。

## v0.0.137 の複数リポジトリ対応がアツイ

直近の変更で個人的に注目しているのが、[v0.0.137](https://github.com/BloopAI/vibe-kanban/releases/tag/v0.0.137-20251216142426) で追加された**複数リポジトリプロジェクト対応**です。

今回の対応により、プロジェクト設定ページで**複数のリポジトリを追加**でき、エージェントが**複数のリポジトリを同時に編集**できるようになりました。

![複数リポジトリの設定](/images/2025-12-17-vibe-kanban/multi-repository-setting.png)

![複数リポジトリがある場合のタスク起動画面](/images/2025-12-17-vibe-kanban/multi-repository-setting.png)

これにより、**フロントエンドとバックエンドで別リポジトリ**だったり、**モノレポにしてなかったがサービスに関連するリポジトリ**を横断して変更できるようになりました！


## まとめ

Vibe-Kanban は AI コーディングエージェントの並列実行を効率化するツールとして、非常に活発に開発が進められています。

特に v0.0.137 で追加された複数リポジトリ対応は、実際のプロダクト開発において大きな価値をもたらす機能だと感じています。毎日のようにリリースされる更新を追いかけつつ、今後の機能拡張にも期待したいところです。

AI ツールを活用した開発効率化に興味がある方は、ぜひ試してみてください。

```bash
npx vibe-kanban
```
